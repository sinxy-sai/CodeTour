# CodeTour

> 基于 [洛谷](https://www.luogu.com.cn/) 的个人算法学习与刷题记录。

[![Language](https://img.shields.io/badge/language-Python%203-3776AB?logo=python&logoColor=white)](https://www.python.org/)
[![Platform](https://img.shields.io/badge/platform-%E6%B4%9B%E8%B0%B7-EA2027)](https://www.luogu.com.cn/)
[![License](https://img.shields.io/badge/license-MIT-green.svg)](LICENSE)

本仓库用于系统整理算法与数据结构题目的 Python 实现。代码按照题型分类存放，文件名通常与洛谷题号一致；部分专题目录同时包含总结笔记，用于记录常见模型、状态设计和解题思路。

## 目录结构

```text
CodeTour/
├─ 动态规划/
│  ├─ 背包/          # 01 背包、完全背包、多重背包等
│  ├─ 区间/          # 区间动态规划
│  ├─ 线性/          # 线性动态规划
│  ├─ 树形/          # 树形动态规划
│  └─ 状压/          # 状态压缩动态规划
├─ 图论/             # 图的遍历、最短路、生成树等
├─ 字符串/           # 字符串匹配与处理
├─ 搜索/             # DFS、BFS、回溯与剪枝
├─ 树形数据结构/     # 树、堆及相关结构
└─ 线性数据结构/     # 栈、队列、链表等
```

目录会随学习进度持续扩充。尚无代码的分类用于后续专题整理。

## 文件规范

- 解题代码使用 Python 3 编写，文件名以题号为主，例如 `P1048.py`、`B2173.py`。
- 同题的补充实现通过简短后缀区分，例如 `P1077_ez.py`。
- 专题知识整理统一命名为 `总结.md`，与对应题目代码放在同一目录。
- 代码以通过在线评测为基本目标，并尽量保留关键状态定义、转移思路或实现说明。

## 本地运行

克隆仓库：

```bash
git clone https://github.com/sinxy-sai/CodeTour.git
cd CodeTour
```

进入相应目录后运行题目代码：

```bash
python "动态规划/背包/P1048.py"
```

程序按照洛谷题目的标准输入格式读取数据，并将答案输出到标准输出。题目要求、输入格式和样例请在洛谷对应题目页面查看，例如：

```text
https://www.luogu.com.cn/problem/P1048
```

## 学习方式

本仓库主要遵循以下整理流程：

1. 按专题学习基础模型与常见变形。
2. 独立完成题目并通过洛谷评测。
3. 将代码归档到对应题型目录。
4. 总结可复用的状态设计、转移方程和易错点。
5. 定期回顾并补充更清晰或更高效的实现。

## 使用说明

仓库内容是个人学习过程的记录，解法不一定是唯一或最优实现。建议先独立思考，再将代码作为思路参考；不同语言版本、Python 版本或评测环境也可能影响运行结果。
